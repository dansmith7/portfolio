# Проекты из админки не отображаются

## Быстрая проверка

### 1. Главная vs страница Work

- **Главная (Latest Works)** — показываются только проекты с галочкой **«Отображать на главной»**.
- **Страница Work** — показываются все проекты.

Проверьте, что у нужных проектов в админке включена опция «Отображать на главной», если ожидаете их на главной.

---

### 2. Подключение к Supabase

Сайт использует данные из Supabase. Если Supabase не подключён, показываются fallback-данные.

**Локально:**
1. В корне проекта должен быть файл `.env`:
   ```
   VITE_SUPABASE_URL=https://ваш-проект.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJ... (anon public ключ)
   ```
2. Перезапустите `npm run dev` после изменения `.env`.

**Vercel (деплой):**
1. Settings → Environment Variables.
2. Добавьте `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY`.
3. Сделайте Redeploy (новый билд обязателен).

Подробнее: [VERCEL-DEPLOY.md](./VERCEL-DEPLOY.md)

---

### 3. Обход кэша

Добавьте `?nocache=1` к URL, чтобы принудительно запросить свежие данные:
- `http://localhost:3000/?nocache=1`
- `https://ваш-сайт.vercel.app/work?nocache=1`

---

### 4. Консоль браузера (F12)

Откройте F12 → Console и проверьте предупреждения:
- `Supabase не подключен` — не заданы или неверные env-переменные.
- `Ошибка загрузки проектов: ...` — проблема с запросом к Supabase.

---

### 5. Network (F12 → Network)

При загрузке главной или /work должны быть запросы к `*.supabase.co`. Если их нет — Supabase не подключён.
